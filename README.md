# Ecommerce Crawler

CLI-based ecommerce product crawler for Indonesian marketplaces.

Supports **Tokopedia** and **Shopee**.

## Features

- Keyword-based product search via marketplace frontend APIs
- Configurable pagination (default: 3 pages)
- Product rating and review count enrichment
- Normalized JSON output with consistent schema
- Retry with exponential backoff for transient failures
- User-agent rotation
- Debug mode to save raw API/HTML responses
- Clear, timestamped logging

## Prerequisites

- **Node.js** >= 20.0.0 (uses native `fetch`; cheerio requires Node 20+)
- **npm**

## Installation

```bash
git clone <repo-url>
cd ecommerce-crawler
npm install
```

## Usage

```bash
node src/main.js --platform <platform> --query <keyword> [options]
```

### Required arguments

| Argument     | Description                              |
|--------------|------------------------------------------|
| `--platform` | Platform to crawl (`tokopedia`, `shopee`) |
| `--query`    | Search keyword                           |

### Optional arguments

| Argument    | Default | Description                          |
|-------------|---------|--------------------------------------|
| `--pages`   | 3       | Number of pages to crawl             |
| `--output`  | auto    | Output JSON file path                |
| `--timeout` | 15000   | Request timeout in ms                |
| `--retries` | 3       | Max retry attempts                   |
| `--debug`   | off     | Save raw responses to debug JSON file |

### Examples

```bash
# Tokopedia — crawl 3 pages of "mouse"
node src/main.js --platform tokopedia --query "mouse" --pages 3 --output output/tokopedia_mouse.json

# Shopee — crawl "mouse wireless"
node src/main.js --platform shopee --query "mouse wireless"

# With debug mode (saves raw responses)
node src/main.js --platform tokopedia --query "laptop" --debug

# Custom timeout and retries
node src/main.js --platform tokopedia --query "keyboard" --timeout 20000 --retries 5

# Output path is auto-generated if omitted
node src/main.js --platform shopee --query "headset"
```

## Output

Products are saved as a JSON array. Each product follows this schema:

```json
{
  "platform": "tokopedia",
  "query": "mouse",
  "page": 1,
  "name": "Logitech Wireless Mouse M185",
  "url": "https://www.tokopedia.com/...",
  "price": 179000,
  "rating": 5,
  "review_count": 3,
  "sold_count": "10rb+ terjual"
}
```

| Field          | Type             | Notes                                          |
|----------------|------------------|-------------------------------------------------|
| `platform`     | string           | Source marketplace                               |
| `query`        | string           | Search keyword used                             |
| `page`         | number           | Page number (1-indexed)                         |
| `name`         | string           | Product name                                    |
| `url`          | string \| null   | Product detail URL                              |
| `price`        | number \| null   | Price in IDR                                    |
| `rating`       | number \| null   | Product rating                                  |
| `review_count` | number \| null   | Total ratings/reviews                           |
| `sold_count`   | string \| null   | Sold count label (Tokopedia only, null on Shopee) |

## Platform Details

### Tokopedia

- **Method**: Direct GraphQL API requests (`SearchProductV5Query`)
- **Products per page**: 60
- **Pagination**: Fully supported via `additionalParams` chaining from response headers
- **Enrichment**: Separate `productRatingAndTopics` endpoint for review counts (5 concurrent requests)
- **Fields from listing**: name, price, rating, URL, sold count
- **Fields from enrichment**: rating (override), review count

### Shopee

- **Method**: HTML DOM scraping via Googlebot user-agent (product cards parsed from rendered HTML)
- **Products per page**: ~40
- **Pagination**: Limited — only page 1 returns product data (Shopee SSR limitation)
- **Fields from search page**: name, price, rating, URL
- **Enrichment**: Each product detail page is fetched for review count via JSON-LD (10 concurrent requests)
- **Why not API?**: Shopee's search API (`/api/v4/search/search_items`) requires anti-bot tokens generated by client-side JavaScript, making direct HTTP requests infeasible without browser automation

## Engineering Decisions

- **Request-based crawling (Tokopedia)**: Uses Tokopedia's GraphQL frontend API directly instead of HTML scraping. Faster, more reliable, and less fragile than DOM parsing.
- **HTML DOM scraping (Shopee)**: Shopee's search API is protected by anti-bot tokens requiring browser-side JavaScript execution. Instead of adding heavy browser automation (Puppeteer/Playwright), the crawler uses Googlebot user-agent to access Shopee's server-side rendered HTML, parsing product cards directly from the DOM (~40 products per page). Review counts are enriched via JSON-LD from individual product detail pages.
- **Two-step enrichment**: Both platforms use a two-step approach — listing retrieval followed by per-product detail/rating enrichment with controlled concurrency.
- **Native fetch**: Uses Node.js built-in `fetch` (available since v18) to avoid unnecessary HTTP library dependencies.
- **ES modules**: Uses `"type": "module"` for clean `import/export` syntax.
- **Minimal dependencies**: Only `commander` for CLI parsing. Everything else uses Node.js built-ins.

## Assumptions

- Tokopedia's GraphQL endpoints remain publicly accessible without authentication
- Shopee continues to render product cards in HTML for search engine crawlers
- `totalRating` from Tokopedia's rating endpoint represents total number of ratings/reviews
- Rating enrichment is best-effort; if it fails for individual products, listing data is preserved
- Price values are in IDR (Indonesian Rupiah)
- Shopee review counts come from product detail page JSON-LD; if enrichment fails, `review_count` is `null`

## Limitations

- **Shopee**: Limited to ~40 products per search (only page 1 renders product cards for bot user-agents)
- **Shopee**: `sold_count` and `review_count` depend on detail page enrichment; `sold_count` is not available
- No proxy support; may be rate-limited under heavy use
- Tokopedia rating enrichment adds ~1 request per product (mitigated by batched concurrency)
- API/query structures may break if platforms change their frontend
